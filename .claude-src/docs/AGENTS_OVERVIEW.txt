The Agents

1. PM Agent (Auggie) - `pm.md`
**Role:** Product Manager & Architect  
**Tool:** Augment Code (superior context engine)  
**Responsibilities:**
- Research codebase patterns
- Create detailed specifications
- Reference existing code patterns
- Define success criteria

2. Developer Agent - `developer.md`
**Role:** Implementation Engineer  
**Tool:** Claude Code  
**Responsibilities:**
- Read specifications
- Implement features following patterns
- Use existing utilities
- Write code that passes tests

3. TDD Agent - `tdd.md`
**Role:** Test-Driven Development Specialist  
**Tool:** Claude Code  
**Responsibilities:**
- Write tests before implementation
- Ensure comprehensive coverage
- Follow Red-Green-Refactor cycle
- Test behavior, not implementation

4. Reviewer Agent - `reviewer.md`
**Role:** Code Quality Gate  
**Tool:** Claude Code  
**Responsibilities:**
- Review implementations
- Check convention adherence
- Flag security/performance issues
- Provide constructive feedback

5. React Specialist - `react.md`
**Role:** React & Tailwind Expert  
**Tool:** Claude Code  
**Responsibilities:**
- Component architecture
- Hooks patterns
- Performance optimization
- Tailwind CSS styling

6. MobX Specialist - `mobx.md`
**Role:** State Management Expert  
**Tool:** Claude Code  
**Responsibilities:**
- Store architecture
- Observable patterns
- Computed values
- Flows and actions

---

Shared Principles (All Agents)

All agents share these 5 core principles:

1. **Investigation First** - Read actual code before making claims
2. **Follow Existing Patterns** - Use what's already there
3. **Minimal Necessary Changes** - Surgical edits only
4. **Anti-Over-Engineering** - Simple solutions, existing utilities
5. **Verify Everything** - Test, check criteria, provide evidence

These principles are repeated at the start of every agent response to maintain consistency.
